# Decision Agent 特性梳理（来源：产品 PPT）

## 定义

Decision Agent 是以**业务知识网络（BKN）**为统一业务语义基础，融合企业多源异构数据、业务系统与自动化流程，通过 **ContextLoader** 动态精准构建高质量上下文，并由 **Dolphin 动态编排引擎**驱动多智能体完成规划（Plan）、推理（Reason）、行动（Act）与工具调用闭环，在全链路可观测、可评测、可审计等机制保障下，借助大语言模型实现 Data+AI 落地企业核心业务场景的**专有 Agent**。

---

## 核心技术栈

| 组件 | 职责 |
|------|------|
| 业务知识网络（BKN） | 统一语义底座：抽象概念/对象/关系/逻辑/行动，实现所有 Agent 全局复用 |
| ContextLoader | 动态按需加载上下文（工具定义降低 90%+），替代粗放向量检索 |
| Dolphin 编排引擎 | 多智能体动态调度；Plan-Reason-Act 状态机；协程调度；可观测 |
| ISF（信息安全编织） | 权限校验、敏感过滤、提示词注入防护、Human-in-the-loop 审批 |
| Autoflow | 跨 Agent 流程编排，节点化复用，无代码配置 |

---

## 五大技术目标

### 1. 降低开发复杂性：语义建模解耦与低代码装配
- BKN 统一语义，消除每个 Agent 单独硬编码业务逻辑的依赖
- Agent 角色、Dolphin 指令、技能配置均支持无代码定义与复用

### 2. 维护与优化自动化：Agent 全生命周期闭环
- "配置-测评-运行-可观测-优化" 自动闭环
- 基于运行轨迹快速定位根因；Benchmark 驱动自动优化

### 3. 高质量上下文：精准装配与按需加载
- 概念召回 → 对象召回 → Pre-Ranking/Re-Ranking
- Memory 按人/岗/任务及 Agent 状态动态装配
- 工具定义上下文占用降低 90%+；执行结果占用降低 80%+

### 4. 多智能体协同：统一语义与动态调度
- Dolphin 实现任务动态分工与自动编排
- 共享 Memory 传递任务状态，降低协作失真
- 状态机管理确保 Plan-Reason-Act 流程可控可追溯

### 5. 安全控制：全链路可审计防御围栏
- RBAC 权限、日志审计（等保/GDPR）、数据脱敏、提示词注入防护
- Human-in-the-loop 审批覆盖关键动作

## 五大业务价值

| 价值维度 | 转变描述 |
|---------|---------|
| 决策提效 | 从"信息生产"到"决策驱动" — 经营分析从人找数/对口径到直接出结论 |
| 知识可信 | 从"口口相传"到"有据可查" — 输出"结论 + 证据链"，来源可追溯 |
| 流程自动化 | 从"人肉串联"到"闭环交付" — 跨部门任务串为 AutoFlow，人只做关键审批 |
| 数据资产可用 | 从"事后救火"到"持续稳定可用" — 自动定位口径冲突、生成治理建议 |
| 风险可控 | 从"不可用的黑盒"到"可审计的生产能力" — 满足合规审计要求 |

## 当前核心痛点（待研究支撑的设计缺口）

1. **多智能体协同失真**：Agent 间输出格式不统一，信息在多轮传递中被压缩、丢关键约束
2. **全局调度/仲裁缺失**：缺少冲突治理机制，多 Agent 容易循环或重复劳动
3. **Worker 独立闭环能力**：当前 Decision Agent workers 是否需要持续依赖上下文管理器（Dolphin/openclaw）才能完成长链任务，尚无明确答案
4. **工具幻觉**：模型越强越容易在工具调用上自信地犯错；BKN 语义解耦是否能有效抑制有待验证
5. **长链规划退化**：Plan-Reason-Act 状态机在 >5 步任务中的规划质量缺乏学术基准支撑

## 架构图关键组件关系

```
Decision Agent 应用层
  └── 各类专用 Agent（数据治理/情报分析/质量探测/运维分析/报告生成...）

Decision Agent Core
  ├── 主控 Agent（Dolphin 驱动 Plan-Reason-Act 状态机）
  ├── ContextLoader（概念召回 → 对象召回 → Re-Ranking）
  ├── 技能/Memory/模型工厂
  └── ISF（安全编织层）

Autoflow（可编排流程复用 + 智能体节点嵌入）

生命周期管理：配置 → 测试 → 发布 → 运行 → 观测 → 优化
```
